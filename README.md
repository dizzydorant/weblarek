# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные:
В этом разделе описаны основные интерфейсы данных, используемые в приложении для обеспечения строгой типизации и корректной передачи информации между модулями.

#### IProduct
Интерфейс описывает структуру объекта товара, представленного в каталоге и карточке товара.

* **id**: `string` — Уникальный идентификатор товара.
* **description**: `string` — Полное текстовое описание характеристик.
* **image**: `string` — Ссылка (URL) на изображение товара.
* **title**: `string` — Краткое название товара.
* **category**: `string` — Категория товара .
* **price**: `number | null` — Цена товара. Значение `null` используется для товаров, которые нельзя приобрести.

**Назначение:** Используется для типизации данных, получаемых от API, и для отрисовки карточек в галерее.

#### IBuyer
Интерфейс описывает данные покупателя, которые собираются через экранные формы при оформлении заказа.

* **payment**: `TPayment` — Способ оплаты (например, 'онлайн' или 'при получении').
* **email**: `string` — Электронный адрес для отправки чека.
* **phone**: `string` — Контактный номер телефона.
* **address**: `string` — Адрес доставки заказа.

**Назначение:** Служит для валидации вводимых пользователем данных и формирования итогового объекта заказа для отправки на сервер.

#### TPayment
Тип-литерал, который ограничивает возможные способы оплаты заказа.

`type TPayment = 'card' | 'cash'`

### Модели данных: 

#### Класс Products
Назначение:
 Управление списком товаров и состоянием предпросмотра.

Зона ответственности:
 Хранение данных, полученных с сервера, поиск товаров и выбор активного товара для модального окна.

Поля класса:
`_items: IProduct[]` - 	Приватный массив всех доступных товаров.
`_preview	IProduct | null` - Данные товара, выбранного для детального отображения.

Методы класса: 
`setItems(items: IProduct[]): void` - Сохраняет массив товаров в модель.
`getItems(): IProduct[]` - Возвращает актуальный массив всех товаров.
`getItem(id: string): IProduct | undefined` - Находит и возвращает один товар по его уникальному идентификатору.
`setPreview(item: IProduct): void` - Записывает товар в поле _preview для последующего открытия модального окна.
`getPreview(): IProduct | null` - Возвращает товар, который сейчас находится в режиме просмотра.

#### Класс Basket
Назначение:
 Управление списком покупок пользователя.

Зона ответственности: 
 Добавление и удаление товаров, расчет итоговых показателей (стоимость,количество).

Поля класса:
 `_items	IProduct[]` -	Массив товаров, добавленных в корзину.

Методы класса:
 `getItems(): IProduct[]` - Возвращает список всех товаров в корзине.
`add(item: IProduct): void` - Добавляет переданный объект товара в корзину.
`remove(id: string): void` - Удаляет товар из массива по его id.
`clear(): void` - Полностью очищает корзину (используется после успешного оформления заказа).
`getTotal(): number` - Вычисляет и возвращает общую сумму всех товаров в корзине.
`getCount(): number` - Возвращает количество элементов в корзине (длину массива).
`inBasket(id: string): boolean` - Проверяет, есть ли уже товар с указанным id в корзине.

#### Класс Buyer
Назначение: 
 Хранение и проверка контактных данных покупателя.

Зона ответственности: 
 Сбор данных из форм оформления заказа и их валидация перед отправкой.

Поля класса: 
`payment: TPayment | null` - Выбранный способ оплаты.
`address: string`	- Адрес доставки.
`email: string`	- Электронная почта пользователя.
`phone: string`	- Телефон пользователя.

Методы класса:
 `setData(data: Partial<IBuyer>): void` - Принимает объект с данными. Обновляет только те поля, которые переданы, сохраняя остальные значения нетронутыми.
`getData(): IBuyer` - Возвращает объект со всеми текущими данными покупателя для формирования итогового заказа.
`clear(): void` - Очищает все поля данных (сброс формы).
`validate(): Partial<Record<keyof IBuyer, string>>` - Проверяет поля на пустоту. Возвращает объект с текстом ошибок. Если поле валидно, оно отсутствует в результирующем объекте.

### Слой коммуникации

#### Класс LarekApi
Принимает в конструктор экземпляр класса `Api` и предоставляет методы для взаимодействия с бэкендом приложения.

**Конструктор:**
- `constructor(api: IApi)` — принимает объект, реализующий интерфейс `IApi`.

**Методы:**
- `getProductList(): Promise<IProduct[]>` — получает массив товаров с сервера.
- `orderProducts(order: IOrder): Promise<IOrderResult>` — отправляет данные о заказе и возвращает результат оформления.

### Слой Представления (View)

#### # Класс Card
*Базовый абстрактный класс для отображения карточки товара. Наследуется от `Component`.*

**Назначение:** 
Отображение базовой информации о товаре (заголовок, цена) и обработка клика по всей области карточки.

**Зона ответственности:** 
* Связь данных объекта товара с элементами разметки.
* Уведомление системы о клике на карточку через колбэк в конструкторе.

**Поля класса:**
* `_title: HTMLElement` — элемент заголовка.
* `_price: HTMLElement` — элемент для вывода цены.

**Методы класса:**
* `set title(value: string): void` — устанавливает текст заголовка.
* `set price(value: number | null): void` — устанавливает цену (если `null` — выводит «Бесценно»).


#### # Класс CardCatalog
*Наследуется от `Card`. Используется для рендеринга товаров в главной галерее (витрине).*

**Поля класса:**
* `_image: HTMLImageElement` — элемент изображения товара.
* `_category: HTMLElement` — элемент тега категории.

**Методы класса:**
* `set category(value: string): void` — устанавливает текст и меняет CSS-класс категории на основе справочника `categoryMap`.
* `set image(value: string): void` — устанавливает источник (`src`) и описание (`alt`) картинки.


#### # Класс CardPreview
*Наследуется от `Card`. Используется для детального отображения товара в модальном окне.*

**Поля класса:**
* `_description: HTMLElement` — элемент с подробным текстом описания.
* `_button: HTMLButtonElement` — кнопка взаимодействия (добавить в корзину).

**Методы класса:**
* `set description(value: string): void` — устанавливает текст описания товара.
* `set price(value: number | null): void` — расширяет базовый метод: если цена отсутствует, блокирует кнопку покупки и меняет текст.
* `set buttonTitle(value: string): void` — позволяет динамически менять текст на кнопке («В корзину» / «Удалить»).


#### # Класс CardBasket
*Наследуется от `Card`. Используется для отображения товара в списке внутри корзины.*

**Поля класса:**
* `_index: HTMLElement` — элемент для вывода порядкового номера товара в списке.
* `_button: HTMLButtonElement` — кнопка удаления товара из корзины.

**Методы класса:**
* `set index(value: number): void` — устанавливает порядковый номер товара.


#### # Класс Basket
*Компонент отображения корзины. Наследуется от `Component`.*

**Зона ответственности:** 
* Управление списком выбранных товаров.
* Отображение итоговой стоимости и состояния кнопки перехода к оформлению.

**Поля класса:**
* `_list: HTMLElement` — контейнер для списка элементов товаров.
* `_total: HTMLElement` — текстовый элемент итоговой суммы.
* `_button: HTMLButtonElement` — кнопка перехода к оформлению заказа.

**Методы класса:**
* `set items(items: HTMLElement[]): void` — заменяет содержимое списка. Если товаров нет — блокирует кнопку оформления.
* `set total(value: number): void` — обновляет значение итоговой стоимости заказа.


#### # Класс Form
*Базовый класс для всех форм. Наследуется от `Component`.*

**Зона ответственности:** 
* Обработка событий ввода (`input`) через делегирование.
* Отображение текстов ошибок и управление активностью кнопки `submit`.

**Поля класса:**
* `_submit: HTMLButtonElement` — кнопка отправки формы.
* `_errors: HTMLElement` — контейнер для текста ошибок.

**Методы класса:**
* `set valid(value: boolean): void` — управляет доступностью кнопки подтверждения.
* `set errors(value: string[]): void` — выводит массив ошибок в текстовый блок.
* `render(state: Partial<IFormState & T>): HTMLElement` — обновляет состояние полей и валидности.


#### # Класс Order
*Наследуется от `Form`. Предназначен для выбора способа оплаты и ввода адреса.*

**Поля класса:**
* `_buttons: HTMLButtonElement[]` — коллекция кнопок выбора способа оплаты.

**Методы класса:**
* `set payment(name: string): void` — переключает активный CSS-класс (`button_alt-active`) для выбранной кнопки оплаты.
* `set address(value: string): void` — устанавливает значение в поле адреса доставки.


#### # Класс Contacts
*Наследуется от `Form`. Предназначен для ввода email и номера телефона.*

**Методы класса:**
* `set email(value: string): void` — устанавливает текст в поле электронной почты.
* `set phone(value: string): void` — устанавливает текст в поле телефона.


#### # Класс Modal
*Универсальный компонент модального окна. Наследуется от `Component`.*

**Зона ответственности:** 
* Управление отображением (открытие/закрытие).
* Служит контейнером для любого переданного контента (форм, превью, корзины).
* Не подлежит наследованию.

**Методы класса:**
* `open(): void` — отображает окно, добавляя класс активности.
* `close(): void` — скрывает окно и очищает его содержимое.
* `set content(value: HTMLElement): void` — заменяет текущую разметку внутри окна новым элементом.



#### # Класс Header
*Управляет верхней частью страницы. Наследуется от `Component`.*

**Поля класса:**
* `counterElement: HTMLElement` — элемент счетчика количества товаров.
* `button: HTMLButtonElement` — кнопка-иконка корзины.

**Методы класса:**
* `set counter(value: number): void` — обновляет число на индикаторе количества товаров в корзине.


#### # Класс Success
*Информационное окно об успешном оформлении заказа. Наследуется от `Component`.*

**Поля класса:**
* `_description: HTMLElement` — элемент для вывода информации о списанной сумме.
* `_close: HTMLButtonElement` — кнопка закрытия окна (возврата к покупкам).

**Методы класса:**
* `set total(value: number): void` — устанавливает текст с итоговой суммой заказа.
